## What is this?

Point your webcam at your hand. Watch magic happen.

This app captures your webcam feed in the browser, streams frames to a Python backend over WebSocket, detects hand landmarks with MediaPipe, and renders real-time visual effects back to the browser вҖ” all at ~20 FPS.

```
Browser в”Җв”Җв”Җв”Җ raw JPEG frames в”Җв”Җв”Җв”Җв–ә FastAPI + MediaPipe
        в—„в”Җв”Җв”Җ annotated frame + в”Җв”Җв”Җв”Җ
              gesture data
```

## Modes

### вңҰ Particles
Colorful physics particles burst from your palm. The more fingers you hold up, the more particles spawn. Gravity and air resistance make them fall naturally.

```
0 fingers  вҶ’  calm trickle
5 fingers  вҶ’  full explosion
```

### вҷ« Music Visualizer
Your hand becomes a conductor. Move it **up/down** to shift frequency (200вҖ“1400 Hz). Raise more fingers to increase the amplitude. A waveform ring pulses around your palm.

```
hand high  вҶ’  high frequency (treble)
hand low   вҶ’  low frequency (bass)
fingers    вҶ’  volume / amplitude
```

### вңҸ Drawing
Leave colorful trails as you move. Each finger count maps to a different color. Close your fist to stop drawing. Trails fade with alpha as they age.

```
1 finger  вҶ’  thin red line
5 fingers вҶ’  thick cyan stroke
fist      вҶ’  pen up
```

### в¬Ў Doctor Strange
Mystical magic circles spin around your hands. Sparks fly from your fingertips. Hold up all **5 fingers** to open a glowing portal.

```
1-2 fingers  вҶ’  orange circles
3 fingers    вҶ’  yellow circles
4 fingers    вҶ’  blue circles
5 fingers    вҶ’  purple + PORTAL вңҰ
```

## Gesture Recognition

| Fingers | Gesture     | Detection Method                        |
|---------|-------------|------------------------------------------|
| 0       | Fist        | All fingertips below PIP joints          |
| 1       | One         | Index tip above PIP                      |
| 2       | Peace       | Index + middle up, ring + pinky down     |
| 3       | Three       | Index + middle + ring up                 |
| 4       | Four        | All except thumb                         |
| 5       | High Five   | All fingers extended                     |
| вҖ”       | OK Sign     | Thumb tip touches index tip (dist < 4%)  |
| вҖ”       | Peace Sign  | Index + middle up, ring down             |

## Quick Start (Local)

### Prerequisites
- Python 3.10, 3.11, or 3.12 вҖ” **not 3.13**
- Node.js 18+

> **Python 3.13 users:** mediapipe 0.10.30+ is installed automatically and the hand landmark model (~8 MB) downloads on first run.

### Setup

```bash
# Clone or unzip the project
cd hand-gesture-app

# One-command setup (creates venv, installs everything)
chmod +x setup.sh start.sh
./setup.sh
```

### Run

```bash
# Starts both backend (port 8000) and frontend (port 3000)
./start.sh
```

Open **http://localhost:3000** вҖ” allow camera access when prompted.

### Manual (two terminals)

```bash
# Terminal 1 вҖ” Backend
cd backend
source venv/bin/activate
uvicorn main:app --reload --port 8000

# Terminal 2 вҖ” Frontend
cd frontend
npm run dev
```

## Deploy Live

### Step 1 вҖ” Push to GitHub

```bash
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/YOUR_USERNAME/hand-gesture-app.git
git push -u origin main
```

### Step 2 вҖ” Backend on Railway

1. Go to [railway.app](https://railway.app) вҶ’ **New Project вҶ’ Deploy from GitHub**
2. Select your repo
3. **Settings вҶ’ Source вҶ’ Root Directory** вҶ’ `backend`
4. **Settings вҶ’ Networking вҶ’ Generate Domain**
5. Copy your URL: `https://your-app.up.railway.app`

> First deploy takes ~3 min (installs mediapipe). On first request the model downloads automatically.

### Step 3 вҖ” Wire frontend to backend

```bash
echo "VITE_WS_URL=wss://your-app.up.railway.app/ws" > frontend/.env.production
git add . && git commit -m "add prod env" && git push
```

### Step 4 вҖ” Frontend on Vercel

1. Go to [vercel.com](https://vercel.com) вҶ’ **New Project вҶ’ Import Git Repo**
2. **Root Directory** вҶ’ `frontend`
3. Add environment variable:
   - Key: `VITE_WS_URL`
   - Value: `wss://your-app.up.railway.app/ws`
4. Deploy вҶ’ live at `https://your-app.vercel.app`

## Cold Start Prevention

Railway's free tier sleeps inactive services. This app handles it two ways:

**Frontend keep-alive** вҖ” `App.jsx` pings `/health` every 25 seconds while the tab is open, keeping the server warm automatically.

**Backend warm-up** вҖ” `main.py` loads MediaPipe at startup (not on first request), so the first user gets a fast response.

**To fully eliminate cold starts** вҖ” go to Railway **Settings вҶ’ Sleeping вҶ’ Disable** (requires Hobby plan, ~$5/mo).

## Troubleshooting

**`mediapipe` install fails**
вҶ’ You likely have Python 3.13. The setup script handles this вҖ” run `./setup.sh` and it will pick Python 3.12 or 3.11 if available on your system.

**Camera not showing**
вҶ’ Allow camera permissions in your browser. On Mac, check System Settings вҶ’ Privacy вҶ’ Camera.

**`setup.sh: No such file or directory`**
вҶ’ You're inside a nested folder from the zip. Run `ls` вҖ” if you see another `hand-gesture-app/` folder, `cd` into it first.

**Backend connects but no frames appear**
вҶ’ Check browser console. The WebSocket may be connecting but the backend may be crashing on the model file. Check terminal 1 for Python errors.

**Railway cold start (slow first connection)**
вҶ’ The frontend auto-pings `/health` every 25s to prevent this. For zero cold starts, disable sleeping in Railway settings.
